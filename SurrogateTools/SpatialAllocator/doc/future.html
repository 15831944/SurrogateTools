9. Future Enhancements

Version 3.0 includes many new enhancements suggested by users. Additional future enhancements under consideration for implementation include:

  • Support Shapefile as an output of OVERLAY mode.
  • Support IoapiFile as an input to OVERLAY mode.
  • Support I/O API Files as inputs and outputs on Windows.
  • Refine the support for allocating an I/O API file to a Shapefile. Currently only the first time step and layer of data will be allocated and output to the Shapefile.
  • Support input file "chunking" in modes other than OVERLAY mode, including the capability to process a grid in parts.
  • Modify srgmerge to take inputs from environment variables when only one surrogate is being merged or gap-filled.
  • Properly allocate numeric values read in from PointFiles - this will require specifying the information about the attribute type somehow.
  • Optionally add an extra surrogate line for counties on the edge of the grid that would allow the value for grid cell "0,0" to represent the amount of the surrogate that fell outside the grid. This would cause surrogates for counties on the edge of the grid to sum to 1.
  • Provide support for reading data from the new ArcGIS database format.
  • Support gap filling and merging in the same execution of srgmerge.
  • Allow the value for OUTSRG= to span multiple lines.
